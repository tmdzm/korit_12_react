## React에서의 typeScrip 기능 이용
### State / Props
- TS를 적용했을 때는 컴포넌트 프롭의 타입을 정의해야 한다. Component의 prop이 JS객체라는 것을 배웠는데, 그러므로 TS를 적용했을 때 프롭의 타입을 정의하기 위해 `type`또는 `interface`를 이용할 수 있다.

```ts
type HelloProps = {
  name : string;
  age : number;
}
```

이상과 같이 types.ts를 통해 자료형들을 전부 명시해둔다.(추후 프로젝트 때도 사용할 예정)
- type export 의 경우 앞에 적어둘것이다.

```tsx
import {HelloProps} from './types.ts'

export default function HelloComponent({name,age} : HelloProps){

  return (
    <h1>
      안녕하세요, {name} 님. 당신은 올해 {age}살이 되었습니다.
    </h1>
  )
}
```
```tsx
import './App.css'
import HelloComponent from './HelloComponent'

function App() {
  return (
    <>
      <HelloComponent name='김영' age = '20' />
    </>
  )
}

export default App

```

- 이상의 코드 라인과 이미지를 참조했을 때 중요한점은 이하와 같다.
1. App.tsx에 빨간줄이 있지만 브라우저에 값이 출력된다. TypeScript가 JSX의 보조이기 때문에 경고만 띄운 상태이기 때문이다.
즉, 브라우저를 켜기전에 잠재적인 오류를 잡아낼 수 있도록 하는 것이 TypeScript의 첫번째 존재 이유에 해당한다.


```ts
type HelloProps = {
  name : string;
  age? : number;
}
```
로 수정하면 `<HelloComponent name='김영'/>`라고 작성하더라도 오류가 발생하지 않는다. 여기서 첫번째 존재 이유와 동일하게, `?`를 붙이지 않더라도 브라우저 자체가 출력이 안되는 일은 없다는 점이다.

- JSX와의 차이점 중에 하나로 React + JS의 경우에는 빨간줄 뜨는 순간에 브라우저 창에 경고창 뜨면서 문법 수정하라고 난리치는 반면에 TS의 경우는 어디까지나 잠재적인 오류를 '브라우저 실행 전'에 포착하는 것이 목표이기 때문에 그것이 JSX상의 문법적 오류가 아니라면 출력이 된다.

```ts
//매개변수가 없는 함수(call1()유형)
export type = {
  name : string;
  age? : number;
  fn : () => void;
}

// 매개변수가 있는 함수 (call3() 유형)
export type = {
  name : string;
  age? : number;
  fn : (msg: string) => void;
}
```

- type.ts 형태로 미리 분리한 예시를 보여줬지만, 추후 수업에서는 Component내에 집어넣었다가 분리하는 과정을 연습하여 일종의 객체지향형태로 작성해보는 과정을 frontend 버전으로 수업할것이다.

- 이상의 경우에서 props 타입을 정의하는 방법에 대해 수업했다. 이번에는 state관련 수업이다.
- `useState()`를 생각해보면 내부의 initialValue가 뭐냐에 따라 추후 상황이 결정된다.

```tsx
//문자열 초기값
const [ name, setName ] = useState(``);
//숫자 초기값
const [ age, setAge ] = useState(0);
//boolean 초기값
const [ isReady, setReady] = useState(false);
```

그 상황에서 `setAge('ten')`이라고 입력하면 오류가 발생했다. 그러면 TS를 배운 저희는 추론할 수 있는게, useState react hook을 사용하게 됐을 때, initialValue에 대한 타입 추론이 일어난다고 생각해볼 수 있다.

```tsx
  const [ age, setAge] = useState(0);

  // initialValue와 다른 매개변수를 넣어서 호출
  setAge('ten'); //

```
와 같이 타입추론을 통한 매개변수가 number 임에도 함수의 매개변수가 string이기 때문에 타입 불일치로 인한 브라우저 return에 실패했다.
근데 react hook은 jsx에서도 있었기 때무ㅡ넹 아까 위의 예시와 달리 아예 결과값이 나오지 않았다.

- 상태의 타입을 '명시적으로' 정의하는 것도 가능하다. 특히 상태를 null 또는 undefined로 초기화하려면 특히 요구된다. '' 안쓰고 null 쓰려면 TS가 필요하다는 의미가 되겠다.

```tsx
const [ message, setMessage ] = useState<string | undefined>(undefined);
```
- 이상의 코드는 금요일에 배웠던 함수에서의 매개변수 타입의 명시적 정의와 방식이 다르다. 그 이유는 useState라는 react hook이 tsx에서만 적용되는 것이 아니기도 하고, 이미 정의된 함수를 호출하여 message / setMessage 배열을 return한 것이기 때문에 `함수명(매개변수1:'string', 매개변수2: number) : void;`와 같은 이전에 배웠던 방법이 아니라 `<>`제네릭을 썼다는 점이다. 이는 ts자체가 아니라 ts와 react가 결합된 tsx 상에서의 문법이라고 보면 된다.

- 근데 이건 상태가 하나짜리일 때에 가깝다. 다수의 타입을 명시해야 한다면 여태까지 해온 것처럼 type / interface를 사용할 수도 있다.

```tsx
type User = {
  id : number;
  name : string;
  age : number;
}

export default Example1() {

  const [ user, setUser ] = useState<User>({});
  //null 이 가능하게 하려면
  const [user, setUser] = useState<User | null>(null);
  return 중략
}
```

## Promise
- 비동기 연산을 처리하는 고전적인 방법은 연산의 성공 또는 실패에 콜백함수를 달아주는것이다. 연산이 성공했을 경우에는 success 함수가 호출되고, 실패하면 failure 함수가 호출되는 방식이 될것이다.(예외처리의 고전적 방법을 떠올리면 된다.) 의사코딩으로 예시 보여주자면.

```tsx
function doAsyncCall(success, failure){
  if(SUCCEED){
    success(resp);
  } else {
    failure(err);
  }
}

function success(response){
  //응답을 가지고 수행하게 되는 로직 작성
}

function failure(error){
  //오류 처리 로직
}

//함수 호출
doAsyncCall(success, failure);
```

- 이상의 예시는 Promise가 도입되기 전 방식이다. Promise는 JS에서 비동기 프로그래밍의 기본 요소에 해당한다.
- Promise의 정의 : 비동기 연산의 결과를 나타내는 JS객체.
- 이를 이용하게 되면 위에서 처럼 성공 실패에 대한 함수들을 일일이 정의할 필요가 없어지기 때문에 비동기 호출을 실행할 때 코드가 단순화되는 장점이 있다. 그리고 콜백이 중첩되지 않기 때문에 callback hell을 겪지 않아도 된다.

참조 : https://developer-haru.tistory.com/57
- 이하의 예시에서 비동기 호출을 수행하고, 응답이 반환되면 then()메서드 내부의 콜백함수가 실행되어 응답을 argument로 받게 된다.

```js
doAsyncCall().then(response => //응답성공이니까 로직 수행)
//success() 함수 부분을 정의하고 호출하는 단꼐가 한줄로 축약
```
then() 메서드는 Promise 객체를 return한다. 그리고 이 Promise는 세 가지 상태중 하나에 해당하게 된다.
1. Pending(대기) : 초기 상태
2. Resolved / Fulfilled(이행) : 작업성공
3. Rejected(거부) : 작업 실패

이하 예시
```js
const myPromise = new Promise((resolve, reject) =>{
  setTimeout(() => {
    resolve('Hello'); // 'Hello'라는 애 가지고 뭔가 할것이다, 정의하지 않았다.
  }, 500);
});
```
- Promise 객체가 생성될 때와 타이머가 실해오디는 동안에 promise는 대기(Pending) 상태에 있다.<br>
500 밀리초가 지나면 "Hello"라는 값으로 resolve 함수가 호출되고, Promise는 이행(resolved) 상태가 된다. 만약에 오류가 발생하게 되면 .then()을 쓰지는 않았지만 Promise상태가 거부(rejected)가 된다.<br>
-> catch()를 통해 해결

- .then() 메서드를 통해 인스턴스를 서로 연결하여 다수의 비동기 작업을 순차적으로 진행시킬 수 잇다. then : 그리고 나서의 의미

```js
doAsyncCall()
  .then(response => ~~)//응답에서 일부데이터만 가지고 오기
  .then(data => //가지고 온 데이터를 가지고 브라우저에 뿌리는 등의 로직)
```

- 오류 발생시에는 Promise에 오류 처리를 추가가능

```js
doAsyncCall()
  .then(response => //로직수행)
  .then(data => //가지고 온 데이터를 가지고 브라우저에 뿌리는 등의 로직)
  .catch(error => console.log(error)) // 근데 then() 수행중에 뭔가 오류나면 콘솔에 오류내용 확인
```

### async / await - 면접때 물어보기좋은 중요한 개념
- 비동기 호출을 처리하는 .then() 보다 더 최신 방법에 해당한다. 애초에 Promise를 기반으로 하는데, 기본적으로 async / await을 이용하려면 await 표현식을 포함할 수 있는 async()함수를 정의해줘야 한다.

```js
const doAsyncCall = async () => {
  const response = await fetch(`http://sampleapi.com`);
  const data = await response.json();
  //그 다음 이상에서 정의된 data 변수를 가지고 뭔가 하면 된다.

};
```
- fetch() 함수 자체도 return 타입이 Promise 객체이다. 그런데 아까 배웠던 것처럼 Promise가 return 되었음에도 .then() 메서드를 사용하는게 아니라 `await` 키워드를 사용했다는 점이 차이가 있다.

- Promise가 처음 도입되었을 때에는 시간적인 차이를 나타내기 위해 then()이라는 접속사를 활요한 메서드를 정의했었지만, 이후 발전 과정에서 기존의 JS 문법으로 표현을 할 필요성을 느낀 개발자들이 `async / await` 키워드를 추가 개발했다.

- .then()을 도입했다면 : chaining methods를 통한 순차 처리를 예시로 코드를 작성
- async/ await을 도입했다면 : 비동기 처리를 하는 함수를 동기적으로 작성하는 코드를 작성

## fetch API 활용
- fetch API는 라이브러리 설치가 필요없다.
- fetch API는 호출하려는 리소스 경로를 필수 argument로 갖는 fetch() 메서드를 가지고 있다.(jsonplaceholder에서 써봤다.) 웹 요청의 경우는 argument로 서비스의 URL이 된다.<br>
그래서 응답을 반환하는 간단한 GET 요청은 이하와 같다.

```tsx
fetch(`https://someapi.com/${id}`)
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(erro => console.log(error));
```
- 이상의 예시에서 첫 번째 .then()문에 전달되는 response는 요청이 성공했는지 확인하는 데 이용할 수 있는 ok 및 status 속성을 포함하는 Promise 객체이다.<br>
응답 상태가 2xx 형식이면 ok가 되고, 속성값은 true가 된다.

```tsx
fetch(`https://someapi.com/${id}`)
  .then(response => {
    if(response.ok){
      //요청 성공 , 2xx에 해당
    }
    else {
      //문제 발생 예시
    }
  })
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(error => console.log(error));
```

POST 요청과 같은 HTTP 메서드를 이용하려면 fetch() 메서드의 두 번째 argument가 필요하다. 두 번째 argument는 여러 요청을 정의할 수 있는 `객체`이다. POST 요청 예시.

```tsx
fetch(`htt[://someapi.com`,{method: 'POST'})
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(erro => console.log(error));
```
- 그런데 두 번째 argument에 헤더를 추가하는 것도 가능하다.(springboot에서 해봤고, postman에서도 수정해봤다.)<br>
이하에서는 우리가 postman에서 딸깍으로 만들었던 갓을 직접 명시하는 코드

```tsx
fetch(`http://someapi.com`, {
  method: 'POST',
  headers: {'Content-Type' : 'application/json'}
  })
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(erro => console.log(error));
```

- 마지막으로 우리가 todolist에서 localStorage 등에 저장하거나 가지고 올 때 stringify나 json등을 썼었다.

```tsx
fetch(`http://someapi.com`,
  {
    method:'POST',
    header: {'Content-Type': 'application/json'},
    body: JSON.stringfy(data)
  })
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(erro => console.log(error));
```

## axios 라이브러리 이용
- 지금 then() 메서드가 Promise에서도 쓰였고 fetchAPI에서도 예시로 작성되었다. 그래서 수강생분들이 처음 수업들을 때 혼란스러워하는 경우가 많은데<br>
.then()대신에 async/await을 도입할 수 있다. 그리고 fetchAPI는 HTTP 메서드들을 처리하기 위한 API로, 이걸 대체하는 라이브러리가 axios에 해당한다.

- 그러면 섞여서 나올 수 있는게 .then() 대신에 async /await은 쓰지만 fetch API를 그대로 쓰고 있을 수도 있고, axios를 쓰지만 .then()을 그대로 쓸수도 있다.

- fetch API의 불편함을 일부 대체하는 외부 라이브러리인 axios를 도입하기 위해서는 터미널에 `npm install axios`가 요구된다.

- axios 라이브러리 내에는 JSON 데이터의 자동 변환을 지원하기 때문에 json()함수다 필요하지 않다.

```jsx
import axios from 'axios';
// 시작점에 어떤 함수인지 명시
axios.get('http://someapi.com') 
.then(response => console.log(response)) //fetch api에서는 response => response.json()이었다.
.catch(error => console.log(error))
```

- POST 요청 예시
```jsx
axios.post('http://someapi.com',{newObject})
  .then(response => console.log(response))
  .catch(error => console.log(error))
```

- 이상이 axios + then() 메서드 예시이다.

```jsx
const response = await axios({

  data: { brand: '현대', model: '소나타'},// 레지스트레이션 넘버는 일부러 안넣었다.
})
```
- 이상의 코드는 일단 async가 정의되어 있지 않기 때문에 그냥 형식의 참고 자료에 해당한다.<br>
보면 axiso에 .get()/ .post()가 아니라 axios 자체에서 argument로 JS 객체 형태를 보내줬다. -> 그리고 json으로 안 바꿨다.<br>
axios 내부에서 알아서 JS 객체를 json으로 stringfy() 해줬다는 것을 알 수 있다.

5. 이상의 JSON 결과값을 확인했을 때, 저희가 목표로하는 기온 / 설명 / icon에 해당하는 값들을 들고 오기 위해서는 JSON의 특정 key의 value의 자료형이 무엇인지를 확인 하는 것이 중요할 것 같다.<br>
예를 들어 temperature의 경우 `main.temp`거나 `main['temp']`로 작성하면 될 것 같다. 하지만 description과 icon의 경우에는 keu-value로 속기 쉬울 수 있지만 weather의 value가 배열이고, 거기의 0번 element가 JS객체라는 것을 확인할 수 있다. 결과적으로 icon의 값을 frontend의 상태에 저장하기 위해서는 `weather[0].icon`이거나 `weather[0]['icon']`이 되어야 한다.

- 첫 렌더링 시에 fetch()를 시도해서 그 값으로 고정시킬 예정이다. 상태가 바뀔때마다 확인하는 것이 아니기 때문에 `useEffect(() =>{},[])`에서 `,[]` 부분이 그 부분을 통제한다.

- 그러면 첫 렌더링 시에 useEffect() 내부의 콜백함수에서 setWeather() 함수를 호출해서 값을 대입해야할것이다. 그리고 useEffect()의 콜백함수에서 fetch()를 수행하고, Promise객체가 나오게 되면 걔를 json으로 바꿔주고, 그 json 값에서 필요한 key-value 조합을 꺼낸 다음에 상태에 저장해야한다.

```jsx
function App() {
  const [weather , setWeather] = useState({
    temp: '',
    desc: '',
    icon: '',
  })

  useEffect(() => {
    //첫 렌더링시 동작하는 부분
    fetch(`https://api.openweathermap.org/data/2.5/weather?q=Busan&units=Metric&APIkey=`)
    .then(response => response.json())
    .then(data => {
      setWeather({
        temp: data.main.temp,
        desc: data.weather[0].description, 
        icon: data.weather[0].icon
      })
    })
    .catch(err => console.log(err));
  },[]);

```

- 특히 useEffect() 부분 전체에 해당하는 부분을 해석해볼만한 가치가 있다. 어디까지가 useEffect()의 영역인지, 어디가 fetch()의 영역인지, 그 다음 then은 어디까지인지 등을 확인 할 수 있다.

6. icon을 확인해보면 json에서도 그렇고 기본적으로 string 자료형이다. 그림이 아니다.<br>
날씨 아이콘을 표시하려면 아이콘 코드 앞에 `https://openweathermap.org/img/wn/`을 표시하고, 맨 마지막에 endpoint로 json의 icon 값을 가지고 와야 한다.

```tsx
function App() {
  const [weather , setWeather] = useState({
    temp: '',
    desc: '',
    icon: '',
  })

  useEffect(() => {
    //첫 렌더링시 동작하는 부분
    fetch(`https://api.openweathermap.org/data/2.5/weather?q=Busan&units=Metric&APIkey=`)
    .then(response => response.json())
    .then(data => {
      setWeather({
        temp: data.main.temp,
        desc: data.weather[0].description, 
        icon: data.weather[0].icon
      })
    })
    .catch(err => console.log(err));
  },[]);

  if(weather.icon) {
      return (
      <>
      <p>기온 : {weather.temp}℃</p>
      <p>설명 : {weather['desc']}</p>
      <p>아이콘 string : {weather.icon}</p>
      <img src={`https://openweathermap.org/img/wn/${weather.icon}@2x.png`} alt="날씨 아이콘" />
      </>
    )
  } else {
    return <h1>Loading...</h1>//img가 느리게 나오니까 같이 출력되게 하기위해서 
  }
}
```
