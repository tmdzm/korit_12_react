## 환경 변수 이용
- CRUD를 사용하게 될때 axios, fetch API 상관없이 'http://localhost:8080/api'를 반복적으로 사용하게 된다.
- 그런데 로컬 호스트가 아니라 배포까지 하게 되면 새로운 ip 주소가 할당될거고, 도메인 구입하면 특정 문자열이 주소로 바뀔것이다. 그러면 배포해서 ip 주소 받으면 그걸로 다 재댕비 하고, 또 DNS를 통해 문자열 주소 받으면 또 재대입하면서 테스트할게 아니라면 변수에다가 로컬호스트를 대입해두고 통제하는게 훨씬 효율적으로 보인다. 그러면 URL값이 변경되었을때 한 곳만 수정하면 된다.
- 비트 프로젝트의 경우 환경변수 이름은 `VITE_`(대문자)로 시작해야한다. 그리고, 환경변수를 저장하는 곳도 지정되어져있다. 

- 루트 폴더에 `.env` 파일을 생성, gitignore에 .env 추가
- src 하위에 api폴더 -> carapi.ts 생성(API를 호출하는 함수를 모듈로 분리)
- 현재 Cralist.tsx내에 return파트도 있고, caching하는 부분도 있고 함수도 있고 해서 되게 어지럽게 되어있다. getCars()
함수는 Carlist.tsx에서 호출만 하고, 함수정의는 분리하겠다는 의미로 받아들이면된다.
- 환경변수를 불러내는 방법이 또 있다. 비트에서 환경변수는 `import.meta.env`라는 접두사를 통해 참조가 가능하다.

### React Router란?
- 정의 : URL을 기반으로 특정 컴포넌트를 렌더링 할지 여부를 정의할 수 있는 방식
- react-router-dom 라이브러리는 라우팅을 구현하는 데 이용되는 컴포넌트들을 제공해준다.

- 사용되는 Component 예시
  1. BrowserRouter : 웹 기반 애플리케이션으로 인한 라우터
  2. Route : 주어진 위치가 일치하면 정의된 컴포넌트를 렌더링 // 이상에서 Route 컴포넌트의 예시를 보여줬는데, element props는 사용자가 path 프롭에 정의된 contact 엔드포인트로 이동할 떄 렌더링 될 컴포넌트를 정의한다. 경로는 현재 위치를 기준으로 상대 경로를 설정한다. 그리고 엔드포인트의 하위 URL에 또 컴포넌트들을 추가로 설정하는 것도 가능하다. 그리고 `*`를 이용하는 것도 가능하다(우리는 Springboot의 SecurityConfig에서 하위 엔드포인테드 `*`를 설정한 적이 있다.) 

예를 들어 `Route path= '/contact/* element={<Contact/>} />`와 같이 사용하게 될 경우, endpoint가 `http://localhost:5173/contact/*`가 된다. <br>
즉 , contact뒤에 아무거나 입력해도 `<Contact/>` 컴포넌트가 출력된다.

또한, `<Route> </Route>` 방식으로도 쓸 수 있다. <br>
components 폴더 하위에 ContactSeoul 컴포넌트 생성 및 h4 태그로 서울이라고 작성해주고 ContactBusan 컴포넌트 생성 및 h4 태그로 부산이라고 작성

// components/carlist.tsx
import { useQuery , useMutation, useQueryClient} from "@tanstack/react-query";
import {getCars, deleteCar} from '../api/carapi'
import { DataGrid, GridCellParams, GridColDef } from "@mui/x-data-grid";
import {  Snackbar } from '@mui/material';
import { useState } from "react";

export default function Carlist(){
  const queryClient = useQueryClient(false);
  const [open,setOpen] = useState();

  const columns: GridColDef[] = [
    {field: 'brand', headerName: '브랜드', width:200,},
    {field: 'model', headerName: '모델', width:200},
    {field: 'color', headerName: '색깔', width:100},
    {field: 'registrationNumber', headerName: '등록번호', width:100},
    {field: 'modelYear', headerName: '제작년도', width:100},
    {field: 'price', headerName: '가격', width:200},
    {
      field: 'delete',
      headerName: '',
      filterable: false,
      disableColumnMenu: true,
      renderCell: (params: GridCellParams) => (//GridCellParams의 아래에 있는 row,
        <button 
          onClick={() => {
            if(confirm(`${params.row.brand}의 ${params.row.color} ${params.row.model}을(를) 삭제하시겠습니까?`))
            mutate(params.row._links.self.href)
          }}
          >
            Delete
          </button>
      )
    }
  ]//배열, 자꾸 헷갈려서 리스트라 적었는데
  const { data, error, isSuccess } = useQuery({
      queryKey: ['cars'],
      queryFn: getCars
  })

  const {mutate} = useMutation(deleteCar,{
    onSuccess: () => {
      setOpen(true);
      queryClient.invalidateQueries({ queryKey: ['cars']}); // 쿼리 무효화, useQuery는 캐싱과 매우 밀접한 연관이 있는데, 캐시에 삭제된 정보가 계속 남아있기 때문에 쿼리를 무효화 할 필요가있다.
      //자동차 삭제 후 실행, 리렌더링 되게 바꾸고 싶다.
    },
    onError: err => {
      console.log(err);
    }
  }) // mutate를 이용해 호출

  if(!isSuccess){
    return <span>Loading ...</span>
  } else if(error){
    return <span> 자동차 데이터를 가져오던 중 오류가 발생했습니다.</span>
  } else {
    return(
      <>
        <DataGrid
          rows={data}
          columns={columns} //car각각이 json형태로 저장된 배열
          disableRowSelectionOnClick={true} // 클릭한 라인이 파란색으로 잡히는걸 막는다.
          getRowId={row=> row._links.self.href}//고유값 집어넣어야함 ,근데 어떻게 이걸 들고 올 수 있는거지
          //row는 매개변수, 인자는 rows에 저장된 data 하나하나
        />
        <Snackbar
          open={open}
          autoHideDuration={2000} 
          onClose={() => setOpen(false)}
          message='해당 차량 정보가 삭제되었습니다.'
        />
        {/**밀리초 라 2000이면 2초다 */}
      </>
    );
  }
}
