## React List

- .map() method를 중심으로 정리할 것.
```jsx
const arr = [1,2,3,4,5] // 배열 선언 및 초기화
const resArr = arr.map(x => x*2);
// arr 배열의 내부 element를 임의로 x로 선언하고 거기에 *2를 수행했다.
그리고 마지막 element까지 이것이 반복
```

즉, java를 배운 기준으로 봤을때는 내부 element를 꺼내는 반복문 내부에 *2를 하는 식이 있다고 보면 된다.

```java
int[] arr = {1,2,3,4,5};
for(int i = 0; i<arr.length; i++>){
  arr[i] = arr[i] * 2;
}
```

근데 JS나 JSX에서는 html 태그까지 합쳐서 작성을 할 필요가 있었기 때문에,
```jsx
export default function MyList() {
  const data = [1,2,3,4,5]

  return(
    <ul>
      {
        data.map((elem,index) => 
          <li key={index}>List Item : {elem*2}</li> // key = {index} 는 고유한 값이어야 한다. index는 고유한 값이 아니므로 권장되지 않음
        )
      }
    </ul>
  )

}
```
- 어제 처음 작성한 `data.map(elem => <li>List Item : {elem*2}</li>)` 로 작성했을 때 f12 눌러서 개발자 도구 켜보면 Each chil in a list should have a unique "key" prop.이라는 경고문이 떴었다. 리액트 list 항목에는 행이 업데이트, 추가, 삭제 되었는지를 감지하는데 이용되는 고유 키가 필요하다. .map() 메서드에서는 두 번째 argument로 index가 존재하는데, 이는 경고 처리를 하는데 이용된다.

- 다만 이상의 방식은 임시방편이다. 목록을 추가, 삭제, 혹은 정렬을 할 경우 에러 가능성이 존재하기 때문에 index를 key로 쓰는 것은 적절하지 않다.
데이터의 고유키가 존재한다면 그걸 이용하는게 좋은데, 우리 같은 경우는 DB에 저장된 각 테이브릥 PK가 될것이다.
그런데 엔티티의 필드 목록을 드러내기 싫다면 프론트에서만 사용하는 임의적인 고유키가 있어야 하는데, 이는 uuid 개념을 검색하고 학슴하면 된다.(오타가 아니라 uuid맞다)

- element가 객체인 배열이라면, 단순 list로 풀어내기 보다는 테이블 형태로 풀어내는 것이 더 적절하다. 속성들이 header / column 명이 되는 방식으로 내부 값들을 보여주면 되니까. 해당 경우 `<li>`로 풀어내는 것이 아니라 `<tr>/<td>`를 사용하는 편이 좋다. 

```jsx


export default function MyTable() {
  const data = [
    {id : 1, brand : '현대' , model:'그랜저'},

  return(
    <>
      <table>

              <td>{car.brand}</td>
              <td>{car.model}</td>
            </tr>)//두 줄 이상이면 중괄호로 감싸야 하지만, enter가 기준인 것은 아니다.
            //이 경우 코드는 한줄로 읽는다.
            //중괄호를 쓰게되면 리턴을 따로 명싱해야한다.
          }

      </table>
    </>
  )
}
```

## React Event
- 리액트에서의 이벤트처리는 DOM 요소 이벤트 처리와 유사하다.(todolist에서 했다)<br>
클릭했을 때 ul에서 li하나를 추가시켜서 읽어왔었다. 차이점은 html 상에서의 DOM 요소 처리는 태그내에 소문자이지만, React 상에서는 camel 케이스로 이용한다는 것(onClick과 onclick)

```jsx
export default function MyComponent() {
  const handleClick = () => {
    console.log('Button pressed!');
  }
  return (
    <button onClick={handleClick}>Press me</button>
    //괄호를 쓰고 안쓰고의 차이, handleClick()는 함수로 retrun이 있다 즉, 함수의 호출 결과를 담는다. 언제 ? 렌더링 될때
    //괄호를 쓰지 않으면 함수자체를 담지만 호출은 하지 않는다. 호출은 이벤트가 발생하고 나서
  );
} 
```

- 이상의 코드에서 주의해야 할 점은 `<button onClick={handleClick()}>Press me</button>`로 작성하면 안된다는 점이다.

- 소괄호 유무가 앱 실행시(즉, 렌더링시)에서의 차이점이 생겼었다.

- ()를 포함했을 경우에는 버튼을 누르기 전에 이미 alert 창이 뜬다는 차이점이 있다.

```jsx
<button onClick={handleClick()}>Press me</button>

<button onClick={handleClick}>Press me</button>
```
- Counter 컴포넌트에서 작성했을 때 주의해야 할 점은 `setCount(preValue => preValue + 1)`으로 쓰면 안되고 `() => setCount(preValue => preValue+1)`으로 작성해야 한다고 했었다. handleClick 역시 동일한 로직으로, 첫 번째 예시가 함수를 onClikc 이벤트에 등록했다면, 두 번째 잘못된 예시의 경우에는 함수의 결과값을 등록한 것이라고 볼 수 있다. 저희는 Java 때부터 method를 호출하기 위해서 메서드명() 방식을 사용해왔다. 그러면 실행 버튼을 눌렀을 때 메서드의 결과값이 변수에 저장됐었다.<br>
즉 리액트 프로그램은 렌더링 되자마자 함수를 호출해 버렸기 때문에 alert이 실행되었고, 그것은 우리가 원하는 구현 방식이 아니었다. onClick 이벤트에 _함수명_ 만 등록해야, 클릭이 일루어졌을 때 함수가 호출된다.

## React Form
- MyForm2.jsx 생성

```jsx
import { useState } from 'react';

export default function MyForm2() {
  const [text, setText] = useState('');// setText는 최신값으로 덮어쓰는 함수

  const handleChange = (event) => {
    console.log(event.target.value);
    setText(event.target.value);
  };

  const handleSubmit = event => {// 하나라서 괄호 생략가능
    event.preventDefault();
    alert(`당신은 ${text}라고 입력하셨네요!`);
  };

  return (
    <form onSubmit={handleSubmit}>
      <input
        type="text"
        value={text}
        onChange={handleChange}
        placeholder='내용을 입력하시오'
      />
      <input type="submit" value="Press Me!" />
    </form>
  );
}
```

- 이상에서 Form처리라고 하긴 했지만 event 처리 관련 하나 더 학습했다. 너무*10 자주 사용하는 input에서의 onChange 이벤트인데 이건 그냥 default 형태이다 보니 별개의 함수로 이상에서처럼 작성하지 않고 inline으로 작성하기도 한다.

- 예시

```jsx
    <form onSubmit={handleSubmit}>
      <input
        type="text"
        value={text}
        onChange={event => setText(event.target.value)}
        placeholder='내용을 입력하시오'
      />
      <input type="submit" value="Press Me!" />
    </form>
```

- 이상에서까지는 기본적으로 form 태그 내에 input이 하나 밖에 없었지만, 회원 가입을 하든, 로그인을 하든 todolist가 아니라면 웬만하면 input field가 복수로 있을것이다. 수업에서의 예시처럼 username/password라는 상태가 있다.

```jsx
const [ username, setUsername ] = useState('');
const [ password, setPassword ] = useState('');

... 중략

<input type="text" onChange={event => setUsername(event.target.value)}>
<input type="password" onChange={event => setPassword(event.target.value)}>

```
와 같이 작성하게 될것이다. 그러면 이 횟수만큼 익명함수를 하나하나 작성해야 하는데, setter 개수가 field 수만큼 있다보니 오타가 날 확률이 높다.
이를 처리하기 위해선 useState()의 initialValue를 JS객체로 선언하는 방법도 있다.
