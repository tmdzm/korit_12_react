### 사용자 입력의 검색어를 이용해 REST API를 호출하게 만들기
  - 참고 : 이번에는 지난 번과 달리 useEffect()를 통해 REST API를 호출하는 것이 불가능하다. 이는 컴포넌트가 최초 렌더링될 때는 사용자가 입력한 키워드를 넣을 수 없기 때문이다.
    - 그런데 억지로 default 값을 만들고 싶다면 최초에 keyword 상태에 initialValue로 'korit_12'같은 것으로 초기화해두면 최초 렌더링시에도 결과값이 나오기는 한다.

  - 버튼을 클릭했을 때, GET 요청이 가야하며 그것이 keyword 상태에 저장되어야 한다.
  이벤트에 대해서는 arrow function을 썻지만, button의 onClikck 이벤트의 경우에는 axios.get(어쩌고url+keyword)해야하고, then()/catch()등 chaining methods가 요구됙 때문에 onClick = 어쩌구는 비효율적
  - handleClick이라는 별개의 함수로 정의

```tsx
import {useState} from 'react';
import axios from 'axios';
import './App.css'

type Repository = {
  id : number; // 고유값을 통해서 나중에 .map() 적용했을 때 사용
  full_name: string;
  html_url: string;
};

function App() {
  const [ keyword, setKeyword] = useState('')
  const [ repodata, setRepodata ] = useState<Repository[]>([]);

  const handleClick = () => {
    axios.get<{items : Repository[]}>(`https://api.github.com/search/repositories?q=${keyword}`)//fetch가 아니다. get요청
    .then(response => setRepodata(response.data.items))//generic안에 객체 분해, items안의 키가 repository 배열
    .catch(err => console.log(err));
  };
  
  return (
        <>
          <input type="text" value={keyword} onChange={(e) => setKeyword(e.target.value)}/>
          <button onClick={handleClick}>검색</button> 
          {repodata.length === 0 ? (
            <p>검색 결과가 없습니다.</p>
          ) : (
            <table>
              <tbody>
                {repodata.map(repo => (
                  <tr key={repo.id}>
                    <td>{repo.full_name}</td>
                    <td>
                      <a href={repo.html_url}>{repo.html_url}</a>
                    </td>
                  </tr>
                ))}
              </tbody>
            </table>)
          }
        </>
  )
}

export default App
```

# React Query Lib 이용

## React Query란
- 서버에서 클라이언트로 데이터를 가져오거나, 동기화 및 업데이트를 쉽게 만들어주고 캐싱 키능을 지원해주면서 비동기 로직을 보다 쉽게 다루게 해주는 리액트 라이브러리이다.

- 참고 : 캐싱(caching)이란 ? 동일한 데이터를 반복해서 가져오는 것을 줄이기 위해 메모리에 저장해두는 행위

## React Query의 사용 이유?
- 서버에서 받아온 데이터는 언제까지고 신선(fresh)한 상태라고 보장할 수 없다. 시간이 흘러서 사용자가 상호작용하여 데이터가 변했을 수도 있고, 서버에서 변경돼서 새로운 정보를 받아와야 할 수도 있다. 클라이언트에서 해당 로직들을 일일이 구현하는 것은 부담될 수 있다. 여기서 React Query는 최신의 데이터를 간편하게 유지하는 다양한 방법을 제공합니다. 그리고 React Hook 처럼 사용이 가능하다.

## useQuery
### 형식
```jsx
import { useQuery } from '@transtack/react-query';

const { data, isLoading, error } = useQuery(queryKey, queryFn, options);
```
- useQuery의 return 값은 이 세가지 말고도 여러 속성값이 있다. 자주 사용하는 것 세개만 적어놨다.

- useQuery는 **비동기 통신을 통해 GET 요청으로 데이터를 불러오는데 사용**된다.
비동기 통신이 끝나면 `isLoading`은 true에서 false로 바뀌고 `data`에 요청한 데이터가 담기게 되며, `error`는 오류 발생 시 return 된다.

### useQuery에 들어가는 argument
1. `queryKey` : useQuery 마다 부여되는 고유 key. 단순 문자열을 할 수 있고 배열로도 가능하다.
2. `queryFn` : 쿼리가 데이터를 요청하는 데 사용하는 함수. fetch나 axios를 사용
- fetch나 axios의 리턴 자료형은 Promise로 고정이다.
- 그걸 fetch API를 사용했다면 response.json()를 통해서 json 형태로 바꿔줬고 axios를 썼다면 내부적으로 알아서 json으로 바꿔줬다.
- Fn은 function의 축약어

3. `options` : 생략가능한 argument로 옵션 추가 시에 사용된다. enable, retry 등이 존재한다.

### 사용 예시 설명
1. api.js 생성
2. main.jsx를 위에 쓴것 처럼 react-query 적용

```jsx
import { useQuery } from '@tanstack/react-query'
import { fetchPosts } from './api'
import './App.css'

function App() {
  const { isLoading, data: posts } = useQuery({
    queryKey : ["posts"], 
    queryFn : fetchPosts
  });


  return (
    <>
      {
        isLoading ? "로딩중..." : 
        posts.map((post, index) => <div>{`${index} : ${post.title}`}</div>)
      }
    </>
  )
}

export default App
```

```js
export const fetchPosts = async () => {
  const res = await fetch('https://jsonplaceholder.typicode.com/posts')
  const data = await res.json;

  return data;
}
```

- 이상의 코드에서 우리가 알 수 있는 점은, React Query를 사용했을 때 처음 비동기 통신을 한 번만 보내기 위해서 useEffect()를 쓸 필요가 없다. 그러니까, useEffect(( => {}, [])); 를 쓸 필요가 없다는 의미이다. 그리고 useEffect()의 결과는 useState()가 필수적이었다. []내에 들어갈 값이 있어야 했다. 그렇다면 useQuery() 세팅을 통해서 useState / useEffect를 둘 다 쓰지 않았다는 의미가 된다. -> 실제 상황에서는 useState/Effect/Query를 다쓰게 될것이다

- fetch를 사용했던 github REST API와의 차이점으로 일단 input 없이 받아올때 고정시켰다. 그 이유는 keyword를 사용하게 되면 기본적으로 useState()가 요구되기 때문이다.

- useQuery 함수의 형식
`const {isLoading, isError, isSuccess} = useQuery({ queryKey : ['repositories'],queryFn : getRepositories})`
- 내부 속성들 정리
  1. queryKey : 쿼리를 위한 고유한 키로 데이터의 캐싱과 다시 가져오기에 이용됨.
  2. queryFn : 데이터를 가져오는 함수이며 Promise를 return해야 한다.

- useQuery Hook이 return하는 query 객체에는 이상과 같은 중요 속성이 있다.(근데 const{} 라고 작성한 것을 봤을 때 **객체구조분해**가 적용되었음을 알 수 있다.)
  1. isLoading : 데이터를 아직 이용할 수 없음.
  2. isError : 쿼리가 오류로 종료됨
  3. isSuccess : 쿼리가 성공적으로 종료, 쿼리 데이터를 사용할 수 있음
- 우리가 data로 정의한 부분은 -> useQuery()의 결과값인 query 인스턴스에는 응답을 return한 데이터가 포함된다.

- 이상의 코드를 다 작성하고 났을 때, 우리는 repodata와 같은 배열 상태를 정의하지 않았다.<br>
즉, 리액트 쿼리가 데이터 관리와 캐싱을 처리하기 때문에 응답 데이터를 저장하기 위한 상태가 필요없다는 것을 알 수 있다.

- repodata를 console.log 찍었을 때 를 생각해보면 거기에 전체 json이 다 들어가 있고, 그중에 일부 데이터(id, full_name, html_url)만 상태에 저장해서 return 할때 브라우저에 풀어줬었다. 하지만 react query를 사용했을 경우는 repodata라는 배열에 있는 Repository 자료형에 정의된 id, full_name, html_url 이외의 다른것들을 정의하지 않고 사용할 수 있다는 점에서 강점이 있다.

## useMutation
### 형식
```jsx
import { useMutation } from `@tanstack/react-query`

const { data, isLoading, mutate } = useMutation(mutationFn, options);
```
- useQuery가 GET 요청으로 데이터를 불러오는 데 사용된다면, useMutation은 ReactQuery를 이용하여 **서버에 데이터 변경 작업을 요청** 할 때 사용된다. <br>
`isLoading`과 `data`는 useQuery와 같이 완료 여부와 데이터를 return하지만, `mutate`는 mutationFn으로 제공된 함수를 작동시키는 트리거 함수에 해당한다.

- useMutation에 들어가는 argument
  1. `mutationFn` : 데이터를 수정하고, 삭제하는데 사용할 함수. fetch / axios를 사용하여 Promise를 return하난 함수가 와야함
  2. `options` : 다양한 옵션이 있는데 4가지만 소개한다.
    - `onMutate` : Promise 객체를 return하는 callback 함수로써 mutation 함수가 실행되기 전에 실행되고 mutation 함수와 동일한 변수가 제공된다.
    - `onError` : 뮤테이션이 실패한 경우 실행할 callback 함수
    - `onSettled` : 성공/실패 상관없이 실행할 callback 함수
    - `onSuccess` : 성공한 경우 실행할 callback 함수

## 전반적인 React Query 정리
- github 때 react-query 적용 버전은 새로고침을 하지 않아도 재포커스가 될 때 리액트 쿼리에 의해 자동으로 다시 불러오기가 된다.(그 시점에 cache를 확인한다.)
- 재포커스란 다른 브라우저에 가거나 다른 프로그램에 갔다가 다시 브라우저를 보게 된 경우

- 근데 이게 매번 일어나면 너무 귀찮아서 `staleTime`이 있는데, 이는 데이터가 최신으로 유지되는 시간을 규정하는 속성이다. 예를 들어서 주식시자으이 겨우 1초에도 몇십만건이 될 수가 있는데 그거 하나하나 반영하려고 하면 계속 리렌더링이 일어나게 될것이다. 그래서 일부 시간동안은 값을 고정시켰다가 그 다음에 한꺼번에 반영하는 것과 동일한 효과이다. 기본값은 0이며, 쿼리 성공 직후 데이터가 최신이 아니라고 보는 것이다. 그래서 데이터가 자주 변경되지 않는다면 staleTime 값을 설정하여 불필요한 다시 가져오기를 방지할 수 있다.(리렌더링도 마찬가지로 줄일 수 잇다.)

- 예시
```jsx
const { isLoading, isError, data} = useQuery({
  queryKey : ['repositories']
  queryFn: getRepositories,
  staleTime: 60 * 1000, // 밀리초 단위
})
```

- cacheTime 이라는 속성도 존재한다. 이는 비활성 상태인 쿼리들이 gabarge collection 대상이 되는 시간을 지정한다. 기본적으로는 5분

* React Query는 내장된 오류 처리, 캐시 무효화와 함께 **데이터 생성, 업데이트 삭제**를 위한 useMutation hook도 존재한다.(useQuery와의 차이점에 주목할 것). axios 예제를 이하에 추가

```jsx
import{useMutation} from `@tanstack/react-query`;
// 여기서부터 사용법
const mutation = useMutation({
  mutationFn: newCar => {
    return axios.post('api/cars', newCar); //아까는 fetch라서 JS객체형태로 다 집어넣음
  },
  onError: (error, variables, context) => {
    //생성, 수정, 삭제시 업데이트 처리 로직
  },
  onSuccess: (data, variables, context) => {
    //변경 성공 시 로직
  }
});
```

- 이상의 예시는 서버에 POST 요청을 전송하고 Promise를 return하는 함수에 해당한다. 즉 지금 우리가 하는 부분은 프론트 엔드에서 application/json 형태로 값을 보내면 backed 찍고 역직렬화해서 Java 형태로 받았닥, DB 찍고 CRUD가 이루어진 다으멩 그 결과값을 backend로 보내고, frontend로 보내서 브라우저에 성공 / 실패/ 리렌더링 등의 결과값을 보여주는 거라고 할 수 있다.

- 이 useMutation은 데이터 변경<sup>Mutation</sup> 처리를 간소화한다.

- mutationFn에서 특정 CUD 요청을 시도하고, onSuccess / onError와 같은 부수 함수를 제공하여, 성공 응답을 바탕으로 UI를 업데이트하거나 성공 메시지를 (toast message형태, alert형태 등) 띄우는 상호작용을 할 수도 있다. onError의 경우에는 연산 중 오류가 발생했을 때 실행될 callback 함수를 지정하는 데 이용된다.

- 호출 방식이 특이했었다. `mutation.mutate(newCar);` : 정의는 위에서 객체 형태로 잘해놓고 호출 못해서 잘 안쓰는 경우가 많았다.

- 그 다음에 DB에서의 변경 사항을 알아서 받아올 수 있돌고 해야한느데 jsonplaceholder에서는 티가 안났다.
